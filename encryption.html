DChat SDK Encryption

Encryption

DChat uses two complementary encryption schemes: asymmetric E2EE for 1-to-1 messages and symmetric sender keys for group messages. Both support automatic forward secrecy via key rotation.

Nodes never see plaintext. They only route opaque encrypted envelopes identified by wallet addresses.

End-to-End Encryption (Direct Messages)

Direct messages use ECIES (Elliptic Curve Integrated Encryption Scheme) on top of Secp256k1 chat keys. The SDK manages key exchange automatically during the contact handshake.

Task<string> EncryptMessageAsync(string message, string recipientPublicKey, string senderDid)

Encrypts a plaintext string with the recipient's Secp256k1 chat public key. Returns a JSON-serialised EncryptedPayloadEnvelope. Increments the sender's message count for rotation tracking.

ParameterTypeDescription
messagestringPlaintext JSON of the MessagePayload.
recipientPublicKeystringHex-encoded Secp256k1 chat public key of the recipient.
senderDidstringDID of the sender used to look up the active chat key pair for the outbound-count increment.
Task<string> DecryptMessageAsync(string encryptedMessage, string senderPublicKey, string recipientDid)

Decrypts an encrypted envelope using the recipient's private chat key. The SDK automatically selects the correct key generation from the rotation chain based on the envelope metadata.

ParameterTypeDescription
encryptedMessagestringJSON-serialised EncryptedPayloadEnvelope.
senderPublicKeystringHex-encoded Secp256k1 public key of the sender.
recipientDidstringDID of the local user used to look up the right private key.

Chat Key Pair

ChatKeyPair is the short-lived E2EE key. It tracks its own rotation state.

PropertyTypeDescription
IdstringGUID unique to this key pair.
OwnerDidstringDID of the owner (did:dchat:username).
PublicKeystringHex-encoded Secp256k1 public key. Safe to share.
PrivateKeystringHex-encoded private key. Never transmitted.

MaxMessageCountlongThreshold for count-based rotation. Default: 10 000.
MaxAgeDaysintThreshold for age-based rotation. Default: 30 days.

NextKeyIdstring?ID of the successor key after rotation.

Key Rotation

Rotation replaces the active ChatKeyPair with a new one, notifying all contacts so they start using the new public key. Old keys are retained for decrypting historical messages.

Task<bool> ShouldRotateKeyAsync(string ownerDid)

Checks the active key's NeedsRotation flag. Call this before sending to proactively trigger rotation when needed.

Task<ChatKeyPair> RotateKeyAsync(string ownerDid, string reason = "scheduled")

Generates a new ChatKeyPair, marks the current one as rotated, links the chain (via PreviousKeyId/NextKeyId), and stores the new key.

ParameterTypeDescription
ownerDidstringDID of the key owner.
reasonstring"scheduled", "policy_triggered", "emergency", or "manual".
Task<bool> SendKeyRotationMessageAsync(string recipientDid, string recipientWalletAddress, string recipientPublicKey, string newChatPublicKey, string reason)

Sends a KeyRotationMessageData to a contact so they can update their cached copy of your public key. Must be called after RotateKeyAsync for each active conversation.

Rotation flow

// 1. Check if rotation is needed before sending
if (await sdk.ShouldRotateKeyAsync(myDid))
{
    // 2. Generate new key pair
    ChatKeyPair newKey = await sdk.RotateKeyAsync(myDid, "policy_triggered");

    // 3. Notify all active contacts
    foreach (var contact in activeContacts)
    {
        await sdk.SendKeyRotationMessageAsync(
            contact.Did, contact.WalletAddress,
            contact.ChatPublicKey, newKey.PublicKey,
            "policy_triggered");
    }
}

Symmetric Encryption (Group Messages)

Group messages are encrypted with a symmetric AES-256-GCM key called a sender key. Each member has their own sender key distributed to all other members individually via E2EE envelopes.

Task<string> EncryptSymmetricAsync(string plaintext, string symmetricKeyBase64)

Encrypts a plaintext string with a base64-encoded AES-256 key. Returns a JSON envelope containing the ciphertext, IV, and auth tag.

Task<string> DecryptSymmetricAsync(string encryptedPayload, string symmetricKeyBase64)

Decrypts a symmetric envelope using the provided AES key. The SDK looks up the correct sender key automatically when processing inbound group messages.
