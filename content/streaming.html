Streaming

The IGrpcStreamingService manages the bidirectional gRPC stream, ACK queue, reconnect logic, and failed-message tracking. You interact with it indirectly through IDChatSDKClient but understanding it helps you build resilient apps.

Methods

Task<bool> StartStreamingAsync(string senderAddress, object keyPair, CancellationToken cancellationToken)

Opens the gRPC channel, authenticates via ADR-036 signature handshake, and starts the read/write loops. Called automatically by ConnectAsync().

Task StopStreamingAsync()

Gracefully completes the outbound write stream, waits for in-flight ACKs to settle, then closes the channel. Called automatically by DisconnectAsync().

Task ProcessQueuedMessagesAsync()

Drains the outbound queue and sends all pending messages over the active stream. Triggered automatically on reconnect. You can call it manually after returning from background to flush any offline-queued messages immediately.

// Called automatically, but you can trigger it manually:
sdk.MessageReceived += async (_, _) => { };
await grpcService.ProcessQueuedMessagesAsync();
Task<bool> SendImmediateAsync(string messageId, string senderId, string receiverId, string content, MessageContentType contentType)

Writes directly to the gRPC stream, bypassing the persistent ACK queue. No retry, no offline buffering. Used exclusively for latency-sensitive call signaling (WebRTC offer/answer/ICE candidates) where a delayed retry would make no sense.

Fire-and-forget. If the stream is unavailable at call time, this returns false and the message is not queued. Do not use for chat messages.

Failed Message Cache

When a message exceeds its ACK timeout, the SDK stores its ID in an in-memory failed-message cache. On reconnect, ProtocolMessageRetryService reads this cache and re-sends the failed messages.

MethodDescription
StoreFailedMessageId(senderDid, messageId)Records a message as failed.
GetFailedMessageIds(senderDid)Returns all failed message IDs for a sender.
ClearFailedMessageIds(senderDid, ids)Removes specific IDs after successful retry.
ClearAllFailedMessagesForSender(senderDid)Clears the entire cache for a sender (e.g., on sign-out).
CancelMessageTimeout(messageId)Cancels the ACK-wait timer for an out-of-band ack.

Events

All events are raised on the thread pool. Marshal to the UI thread as required by your framework.

OnMessageReceived EventHandler<ChatMessage> Fires when a new inbound message arrives from the stream after being decrypted.
OnMessageSent EventHandler<QueuedMessage> Fires when a message has been written to the stream (before ACK).
OnMessageAcknowledged EventHandler<QueuedMessage> Fires when the server confirms delivery of a sent message.
OnMessageFailed EventHandler<QueuedMessage> Fires when a message times out waiting for an ACK. The message is moved to the failed cache for retry.
OnStreamError EventHandler<Exception> Fires on an unrecoverable stream-level error before reconnect is attempted.
OnConnected EventHandler Fires when the stream is successfully established or re-established after a disconnect.
OnDisconnected EventHandler Fires when the stream closes (graceful or error).
OnReconnecting EventHandler<int> Fires at each reconnect attempt. The argument is the 1-based attempt number. Use for UI indicators.

Subscribing to events

// Subscribe to message events via MessageTypeManager (static)
using DChatSDK.Messaging.Services;

MessageTypeManager.TextMessageReceived += OnTextMessage;
MessageTypeManager.ImageMessageReceived += OnImageMessage;

// Connection events are on the SDK client
sdk.ConnectionChanged += OnConnectionChanged;

Offline Behaviour

When the stream is unavailable, GrpcStreamingService stores outbound messages in an in-memory IQueueService. On reconnect, ProcessQueuedMessagesAsync() drains the queue automatically. The queue is not persisted to disk by default if your app is killed while offline, consider persisting the queue yourself and re-enqueuing on next launch.

Subscribe to OnConnected and call sdk.ConnectAsync() from your app's network-reachability callback to ensure messages are flushed promptly when connectivity returns.
