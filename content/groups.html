Groups

Group chats use symmetric sender keys for efficient encryption and a cryptographic state snapshot system to keep members in sync no central server required.

Architecture

Group Chat Lifecycle

GroupCreateMessageData broadcast
SenderKey Distribution
E2EE envelope to each member
Members Store Key
Ready to decrypt group messages
Send Group Message
AES-GCM(senderKey) broadcast
Member Change
SenderKey rotation triggered
Old Key Invalidated
Removed members lose access

Creating a Group

Groups are created by an admin who broadcasts a GroupCreateMessageData to all initial members. The SDK handles sender-key generation and distribution automatically.

Two Layers: Your app typically wraps the SDK with a service layer (e.g., GroupService) that handles local database operations. The SDK handles the cryptographic protocol and network communication.

SDK-Level Creation (Low-Level)

Direct SDK usage for maximum control:

Step 1: Generate a Group ID

The SDK generates a unique group ID with the proper format.

string groupId = sdk.GenerateGroupId();
// Returns "dc" prefixed ID, e.g., "dc1a2b3c4d5e6f..."
Step 2: Build the Member List

Create GroupMemberInfo objects for each member. You need their DID, wallet address, and chat public key (obtained during contact exchange).

var members = new List<GroupMemberInfo>
{
    new GroupMemberInfo
    {
        Did = "did:dchat:bob",
        WalletAddress = "dc1qbob...",
        ChatPublicKey = "03a1b2c3...",  // from contact exchange
        DisplayName = "Bob"
    }
};
Step 3: Create and Send

The creator is automatically added as admin. SDK generates sender keys and distributes them encrypted to each member.

var createData = new GroupCreateMessageData
{
    GroupId = groupId,
    GroupName = "Project Team",
    Members = members,
    Admins = new List<string> { yourDid }  // you are admin
};

bool success = await sdk.SendGroupCreateAsync(createData, yourDid);

Service Layer Pattern (Recommended)

In production apps, wrap the SDK with a service that handles local state:

// Your app's GroupService wraps the SDK
public class GroupService
{
    private readonly IDChatSDKClient _sdk;
    private readonly IGroupRepository _repo;

    public async Task<ConversationId.Group> CreateGroupAsync(
        string groupName,
        string[] memberDids,
        string creatorDid)
    {
        // 1. SDK generates the group ID
        var groupId = _sdk.GenerateGroupId();

        // 2. Save to local database FIRST
        await _repo.CreateAsync(new GroupEntity
        {
            Id = groupId,
            GroupName = groupName,
            CreatorDid = creatorDid
        });

        // 3. Add members to database
        foreach (var did in memberDids)
            await _repo.AddMemberAsync(groupId, did);

        // 4. SDK broadcasts to network
        await _sdk.SendGroupCreateAsync(createData, creatorDid);

        return new ConversationId.Group(groupId);
    }
}
Why save first? Saving to your local database before sending ensures the group exists even if network fails. The SDK handles retries for the network message.
Full API Reference: See Group Control API for all group management methods (add/remove members, change name, etc.).

Detecting Groups

When you receive a message, you need to know if it's for a group or direct chat. The SDK delegates this check to your app via IGroupMemberProvider.

How It Works

Message Routing Flow
Incoming Message
Message arrives with ChatId field
SDK Calls IsGroupAsync(ChatId)
Delegates to your IGroupMemberProvider
true = Group
Use group decryption with sender key
false = Direct
Use ECDH decryption with chat key

Implementing IGroupMemberProvider

You must implement this interface and register it during SDK initialization. The SDK doesn't store group state - your app does.

public class GroupMemberProvider : IGroupMemberProvider
{
    private readonly IGroupRepository _repo;

    public async Task<bool> IsGroupAsync(string chatId)
    {
        // Check YOUR database - SDK doesn't know
        var group = await _repo.GetByIdAsync(chatId);
        return group != null;
    }

    public async Task<IReadOnlyList<GroupMemberInfo>> GetGroupMembersAsync(string groupId)
    {
        // Return all active members for sender key distribution
        return await _repo.GetMembersAsync(groupId);
    }
}

Registering the Provider

var options = DChatSdkOptions.CreateBuilder()
    .AddNode(nodeConfig)
    .SetGroupMemberProvider(new GroupMemberProvider(repo))
    .Build();

Convenience Method

IsGroupChatAsync Task<bool>

SDK wrapper that calls your IGroupMemberProvider.IsGroupAsync internally.

bool isGroup = await sdk.IsGroupChatAsync(chatId);

if (isGroup) {
    // Route to group chat UI
} else {
    // Route to direct message UI
}
Why delegate to your app? The SDK is stateless by design. Your app manages the UI and local database, so only you know which chat IDs are groups. This separation keeps the SDK focused on cryptography and networking.

Handling Incoming Group Events

When someone adds you to a group, the SDK receives a GroupCreate message and fires an event. Your app must listen and update its local state.

Step 1: Subscribe to Group Events

The SDK's MessageTypeManager fires events for all group protocol messages. Subscribe during app initialization:

// In your service initialization (e.g., MessageStreamingService)
using DChatSDK.Messaging.Services;

public void Initialize()
{
    // Group lifecycle events
    MessageTypeManager.GroupCreateReceived += HandleGroupCreateReceived;
    MessageTypeManager.GroupMemberAddReceived += HandleGroupMemberAddReceived;
    MessageTypeManager.GroupMemberRemoveReceived += HandleGroupMemberRemoveReceived;
    MessageTypeManager.GroupAdminUpdateReceived += HandleGroupAdminUpdateReceived;

    // Group settings events
    MessageTypeManager.GroupNameChangeReceived += HandleGroupNameChangeReceived;
    MessageTypeManager.GroupPhotoChangeReceived += HandleGroupPhotoChangeReceived;
}

Step 2: Implement the Handler

When you receive a GroupCreate event, save the group to your database so it appears in the chat list:

private async void HandleGroupCreateReceived(object sender, GroupCreateReceivedEventArgs args)
{
    var groupId = args.GroupId.RawValue;

    // 1. Skip if already exists (duplicate message)
    if (await _groupRepo.GetByIdAsync(groupId) != null)
        return;

    // 2. Create group entity in your database
    var group = new GroupEntity
    {
        Id = groupId,
        GroupName = args.GroupName,
        GroupDescription = args.GroupDescription,
        CreatorDid = args.CreatorDid,
        CreatedAt = args.Timestamp,
        KeyGeneration = 1,
        StateVersion = 1
    };
    await _groupRepo.CreateAsync(group);

    // 3. Add all members
    foreach (var memberDid in args.InitialMembers)
    {
        var profile = args.MemberProfiles.GetValueOrDefault(memberDid);
        var isAdmin = args.AdminList.Contains(memberDid);

        await _groupRepo.AddMemberAsync(new GroupMemberEntity
        {
            GroupId = groupId,
            MemberDid = memberDid,
            MemberName = profile?.DisplayName ?? memberDid,
            MemberChatPublicKey = profile?.PublicKey ?? "",
            Role = isAdmin ? GroupRole.Admin : GroupRole.Member,
            IsActive = true
        });
    }

    // 4. Create chat entity so group appears in chat list
    await _chatRepo.CreateChatAsync(new ChatEntity
    {
        Id = groupId,
        IsGroupChat = true,
        GroupName = args.GroupName,
        DisplayNameOrGroupName = args.GroupName
    });

    // 5. Fire UI event to refresh chat list
    _eventBus.Publish(new GroupCreatedEvent { GroupId = groupId });
}

Step 3: Handle Member Changes

private async void HandleGroupMemberAddReceived(object sender, GroupMemberAddReceivedEventArgs args)
{
    // Add new member to database
    await _groupRepo.AddMemberAsync(new GroupMemberEntity
    {
        GroupId = args.GroupId.RawValue,
        MemberDid = args.NewMemberDid,
        MemberName = args.MemberDisplayName,
        Role = GroupRole.Member,
        IsActive = true
    });

    // Create system message: "Bob was added to the group"
    await _messageRepo.CreateSystemMessageAsync(
        args.GroupId.RawValue,
        $"{args.MemberDisplayName} was added to the group");
}

private async void HandleGroupMemberRemoveReceived(object sender, GroupMemberRemoveReceivedEventArgs args)
{
    // Mark member as inactive
    await _groupRepo.RemoveMemberAsync(args.GroupId.RawValue, args.RemovedMemberDid);

    // Create system message: "Bob was removed from the group"
    await _messageRepo.CreateSystemMessageAsync(
        args.GroupId.RawValue,
        $"{args.RemovedMemberDid} was removed from the group");
}

Group Messaging

Task<bool> SendGroupMessageAsync(ConversationId.Group groupId, MessagePayload payload, string senderDid)

Sends any typed payload to all group members using sender-key encryption. For text messages, construct a MessagePayload containing a TextMessageData and pass it here.

var payload = new MessagePayload
{
    ContentType = MessageContentType.Text,
    Data = new TextMessageData { Content = "Hello group!" }
};
await sdk.SendGroupMessageAsync(new ConversationId.Group(groupId), payload, senderDid);
Task<bool> SendGroupImageMessageAsync(...)

Uploads the image to IPFS, then distributes an ImageMessageData payload symmetrically to all members.

Task<bool> SendGroupVideoMessageAsync(...)

Uploads video to IPFS and sends a VideoMessageData symmetrically to all members.

Task<bool> SendGroupAudioMessageAsync(...)

Uploads audio and sends an AudioMessageData symmetrically.

Task<bool> SendGroupFileMessageAsync(...)

Uploads a file and sends a FileMessageData symmetrically.

Task<byte[]> DownloadGroupMediaAsync(...)

Downloads and decrypts a media file from IPFS for a group message. The SDK fetches the appropriate sender key from secure storage and decrypts the IPFS payload.

Implementing IGroupMemberProvider

The SDK has no database - it delegates group-membership queries to your application via IGroupMemberProvider. You must implement this interface and pass it via SetGroupMemberProvider() in the options builder.

public class GroupMemberProvider : IGroupMemberProvider
{
    private readonly IGroupRepository _groupRepository;
    private readonly IUserRepository _userRepository;

    public GroupMemberProvider(
        IGroupRepository groupRepository,
        IUserRepository userRepository)
    {
        _groupRepository = groupRepository ?? throw new ArgumentNullException(nameof(groupRepository));
        _userRepository = userRepository ?? throw new ArgumentNullException(nameof(userRepository));
    }

    public async Task<bool> IsGroupAsync(string chatId)
    {
        if (string.IsNullOrEmpty(chatId))
            return false;

        var group = await _groupRepository.GetByIdAsync(chatId);
        return group != null;
    }

    public async Task<IReadOnlyList<GroupMemberInfo>> GetGroupMembersAsync(string groupId)
    {
        var members = await _groupRepository.GetGroupMembersAsync(groupId);
        var memberInfoList = new List<GroupMemberInfo>();

        foreach (var member in members.Where(m => m.IsActive))
        {
            var memberInfo = await GetMemberInfoAsync(member.MemberDid);
            if (memberInfo != null)
                memberInfoList.Add(memberInfo);
        }

        return memberInfoList;
    }

    public async Task<GroupMemberInfo> GetMemberInfoAsync(string memberDid)
    {
        var user = await _userRepository.GetUserByDidAsync(memberDid);

        if (user == null || string.IsNullOrWhiteSpace(user.WalletAddress))
            return null;

        var displayName = !string.IsNullOrEmpty(user.Nickname) ? user.Nickname
            : user.DisplayName ?? user.Username ?? memberDid[..8];

        return new GroupMemberInfo
        {
            Did           = memberDid,
            WalletAddress = user.WalletAddress,
            ChatPublicKey = user.PublicKey,
            DisplayName   = displayName
        };
    }
}

Group State Snapshots

When an admin adds or removes a member, or rotates a sender key, they broadcast a GroupStateSnapshot to bring new members up to speed. The snapshot is hashed, signed, and verified before being applied.
