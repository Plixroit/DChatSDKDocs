Messaging

Send text, images, video, audio, files, albums, and typed protocol payloads. Every message is encrypted before leaving the device and routed via the gRPC stream.

MessagePayload

MessagePayload is the universal message container. It wraps typed MessageDataBase and is serialised to JSON, encrypted, and transmitted as an opaque envelope.

PropertyTypeDescription
MessageIdstringUUID uniquely identifying this message.
ChatIdstringDeterministic hash of participant DIDs (1-to-1) or group ID. Never visible to the server.
TypeMessageContentTypeDiscriminator used to deserialise Data to the correct concrete type.
DataMessageDataBaseThe typed message body (e.g., TextMessageData, ImageMessageData).
ReplyTostring?Message ID of the message being replied to.
IsForwardedboolTrue when this message was forwarded from another conversation.
OriginalMessageIdstring?Source message ID if forwarded.
ForwardedFromDidstring?DID of the original sender.
ExpiresAtlong?Unix timestamp after which the message should be deleted by the recipient.

Delivery Flow

Message Delivery Pipeline
SendTextMessageAsync()
App initiates message send
MessagePayloadFactory
Builds MessagePayload structure
CryptoService
ECIES / AES-GCM encryption
GrpcStreamingService
Enqueues QueuedMessage
gRPC Stream
Writes to server, routes to recipient
Server ACK
OnMessageAcknowledged event
OnMessageReceived
Recipient decrypts and receives

If the stream is offline, messages stay in the in-memory queue and are flushed automatically on reconnect via ProcessQueuedMessagesAsync().

Typed Send Methods

These are convenience wrappers that build the correct MessagePayload, handle IPFS upload for media, and call SendMessageToWalletAsync internally.

Task<bool> SendTextMessageAsync(...)

Sends a plain-text message with optional reply support.

ParameterTypeDescription
recipientWalletAddressstringBech32 address used for gRPC routing.
contentstringUTF-8 message text.
senderDidstringDID of the sender.
recipientDidstringDID of the recipient.
chatIdstringConversation identifier.
recipientPublicKeystringRecipient's chat public key (hex).
messageIdstring?Optional explicit message ID. Auto-generated if null.
replyToMessageIdstring?Message ID this is replying to.
await sdk.SendTextMessageAsync(
    recipientWalletAddress: "dc1qbob...",
    content:                "Hello!",
    senderDid:              "did:dchat:alice",
    recipientDid:           "did:dchat:bob",
    chatId:                 chatId,
    recipientPublicKey:     bobChatPublicKey);
Task<bool> SendImageMessageAsync(...)

Uploads the image and optional thumbnail to IPFS, then sends an ImageMessageData containing the IPFS CIDs and dimensions. Upload progress is reported via the optional IProgress<double>.

ParameterTypeDescription
imageDatabyte[]Raw image bytes (JPEG, PNG, WEBP, etc.).
thumbnailDatabyte[]?Optional pre-generated thumbnail. SDK generates one if null.
width / heightintImage dimensions in pixels.
captionstring?Optional image caption.
uploadProgressIProgress<double>?Reports upload progress (0.01.0).
Task<bool> SendVideoMessageAsync(...)

Uploads the video to IPFS and sends a VideoMessageData with the CID, thumbnail, duration, and dimensions.

ParameterTypeDescription
videoDatabyte[]Encoded video bytes (MP4 recommended).
thumbnailDatabyte[]?Optional video thumbnail.
durationSecondsintDuration of the video.
filenamestringOriginal file name (e.g., "clip.mp4").
Task<bool> SendAudioMessageAsync(...)

Uploads an audio clip and sends an AudioMessageData with the CID, waveform, and duration.

Task<bool> SendFileMessageAsync(...)

Uploads any file to IPFS and sends a FileMessageData containing the CID, filename, MIME type, and file size.

Task<bool> SendAlbumMessageAsync(...)

Uploads multiple images in parallel and sends them as a single AlbumMessageData. Each image is represented by an AlbumImageInput.

var images = new[]
{
    new AlbumImageInput { ImageData = img1, Width = 1080, Height = 720 },
    new AlbumImageInput { ImageData = img2, Width = 800,  Height = 600 }
};
await sdk.SendAlbumMessageAsync(recipientWallet, images, senderDid,
    recipientDid, chatId, recipientPublicKey);

Low-Level Send

Use SendMessageToWalletAsync when you have already built a MessagePayload with custom data.

Task<bool> SendMessageToWalletAsync(string recipientWalletAddress, MessagePayload payload, string recipientPublicKey)

Encrypts payload with the recipient's chat key and enqueues it for gRPC delivery. Returns true if successfully enqueued (not yet delivered listen for OnMessageAcknowledged for delivery confirmation).

Task<bool> SendEncryptedEnvelopeToWalletAsync(string recipientWalletAddress, string encryptedEnvelopeJson)

Sends a pre-encrypted envelope directly. Use when you've handled encryption yourself (e.g., when forwarding a group sender-key distribution). The SDK does not re-encrypt the payload.

Profile & Contact Messages

Task<bool> SendProfileRequest(...)

Sends a ProfileRequestMessageData asking the recipient to share their current profile (avatar, display name, chat public key). Used during contact resolution when the local cache is stale.

Task<bool> SendProfileUpdateAsync(...)

Sends a ProfileUpdateMessageData to a contact with the local user's current profile info. Sent proactively when a profile field changes, or in response to a ProfileRequest.

ParameterTypeDescription
usernamestringCurrent username.
displayNamestring?Optional display name override.
statusMessagestring?Optional status text.
avatarIpfsCidstring?IPFS CID of the avatar image.
senderChatPublicKeystring?Current chat public key so the recipient can encrypt future messages correctly.
updatedFieldsstring[]?Array of field names that changed, enabling partial updates.
requestIdstring?ID from the originating ProfileRequest, if responding.
