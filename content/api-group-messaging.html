Group Messaging Operations

Send messages to groups using sender-key encryption for efficient E2EE.

Overview

Group messages use sender-key encryption: each member has their own symmetric key that they distribute to all other members via E2E encrypted envelopes. This allows efficient broadcast encryption without re-keying on member changes.

Sender-Key Encryption Flow
1. Sender Generates Sender Key
Each member creates a unique symmetric key for their messages
E2E Encrypted
Member A
Member B
Member C
Single Encrypt
2. Broadcast to All Members
One encryption, multiple recipients - efficient group messaging
ConversationId.Group - All group methods take a ConversationId.Group typed identifier, not a plain string. Construct one with new ConversationId.Group(groupId) or use the value directly if you already have one.

SendGroupAlbumMessageAsync Task<bool>

Sends a multi-image/video album to a group using sender-key encryption. Each item is encrypted and uploaded to IPFS in parallel, then a single album message is sent containing all CIDs and thumbnails. Bypasses group detection — explicit group routing guaranteed.

ParameterTypeDescription
groupIdConversationId.GroupTyped group identifier
itemsList<AlbumImageInput>List of album items
senderDidstringSender's DID
captionstring?Optional caption (default: null)
messageIdstring?Optional client message ID (default: null)
onItemProgressAction<int, float>?Called per tile as it uploads: (tileIndex, 0.0–1.0). Default: null
itemCancellationTokensCancellationToken[]?Per-tile cancellation tokens; cancel one to skip that item. Default: null
await sdk.SendGroupAlbumMessageAsync(
    new ConversationId.Group(groupId), items, senderDid,
    caption: "Our trip",
    onItemProgress: (idx, pct) => UpdateTileProgress(idx, pct)
);
