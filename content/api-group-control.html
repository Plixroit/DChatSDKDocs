Group Control Operations

Create groups, manage members, update settings, and handle group metadata.

Setup

The SDK handles cryptography and networking, but your app must subscribe to events to save incoming group messages to your database.

Subscribe to Group Events

During your service initialization, subscribe to MessageTypeManager events:

// In your service initialization
using DChatSDK.Messaging.Services;

public void Initialize()
{
    // Group lifecycle events
    MessageTypeManager.GroupCreateReceived += HandleGroupCreateReceived;
    MessageTypeManager.GroupMemberAddReceived += HandleGroupMemberAddReceived;
    MessageTypeManager.GroupMemberRemoveReceived += HandleGroupMemberRemoveReceived;
    MessageTypeManager.GroupAdminUpdateReceived += HandleGroupAdminUpdateReceived;

    // Group settings events
    MessageTypeManager.GroupNameChangeReceived += HandleGroupNameChangeReceived;
    MessageTypeManager.GroupPhotoChangeReceived += HandleGroupPhotoChangeReceived;
}
Why events? The SDK is stateless - it doesn't have your database. When a GroupCreate message arrives, the SDK decrypts it and fires an event. Your app handles the event and saves to database so the group appears in the chat list.
GenerateGroupId string

Generates a unique group ID prefixed with "dc". Call this before creating a group.

string groupId = sdk.GenerateGroupId(); // e.g., "dc1a2b3c..."

Group Creation

Concept Guide: For a detailed explanation of group architecture and event handling, see Groups.

Complete Creation Flow

// 1. Generate a unique group ID
string groupId = sdk.GenerateGroupId();
// Returns "dc" prefixed ID, e.g., "dc1a2b3c4d5e..."

// 2. Build member info list (you need their public keys from contact exchange)
var members = new List<GroupMemberInfo>
{
    new GroupMemberInfo
    {
        Did = "did:dchat:bob",
        WalletAddress = "dc1qbob...",
        ChatPublicKey = "03a1b2c3...",  // hex Secp256k1 key
        DisplayName = "Bob"
    },
    new GroupMemberInfo
    {
        Did = "did:dchat:charlie",
        WalletAddress = "dc1qcharlie...",
        ChatPublicKey = "03x9y8z7...",
        DisplayName = "Charlie"
    }
};

// 3. Define admins (creator is automatically admin)
var admins = new List<string> { "did:dchat:alice" };

// 4. Create the payload
var createData = new GroupCreateMessageData
{
    GroupId = groupId,
    GroupName = "Project Team",
    GroupDescription = "Development team chat",
    Members = members,
    Admins = admins,
    GroupAvatar = ""  // Optional: base64-encoded image
};

// 5. Send to all members (SDK encrypts sender keys automatically)
bool success = await sdk.SendGroupCreateAsync(createData, "did:dchat:alice");
Where do public keys come from? Member public keys are obtained during contact exchange. When a user accepts a contact request, their public key is shared and stored. Use your app's contact repository to retrieve them.

Method Reference

SendGroupCreateAsync Task<bool>

Sends a group creation message to all initial members with encrypted sender keys. The SDK automatically generates a sender key for the creator and distributes it encrypted to each member.

ParameterTypeDescription
createDataGroupCreateMessageDataGroup creation payload (see below)
senderDidstringCreator's DID

GroupCreateMessageData Properties

PropertyTypeDescription
GroupIdstringUnique group ID from GenerateGroupId()
GroupNamestringDisplay name for the group
GroupDescriptionstring?Optional group description
MembersList<GroupMemberInfo>Initial member list with public keys
AdminsList<string>DIDs of members with admin privileges
GroupAvatarstring?Optional base64-encoded avatar image

GroupMemberInfo Properties

PropertyTypeDescription
DidstringMember's DID (did:dchat:username)
WalletAddressstringBech32 wallet address for message routing
ChatPublicKeystringHex-encoded Secp256k1 public key for encryption
DisplayNamestring?Human-readable name (UI only)

ParameterTypeDescription

SendGroupAdminUpdateAsync Task<bool>

Sends a group admin update message (promote/demote) to all members.

ParameterTypeDescription
groupIdConversationId.GroupTyped group identifier
requesterDidstringRequester's DID
targetMemberDidstringTarget member's DID
promoteToAdminboolTrue to promote, false to demote
stateVersionintCurrent state version
memberDidsstring[]All member DIDs for broadcast
await sdk.SendGroupAdminUpdateAsync(
    new ConversationId.Group(groupId), requesterDid, targetMemberDid,
    promoteToAdmin: true, stateVersion, memberDids
);

RecoverGroupMetadataAsync Task<bool>

Attempts to recover group metadata using fallback strategies.

ParameterTypeDescription
groupIdConversationId.GroupTyped group identifier
requesterDidstringRequester's DID
lastStateCidstring?Last known state CID (default: null)
knownMemberDidsList<string>?Known member DIDs (default: null)
knownAdminDidsList<string>?Known admin DIDs (default: null)
await sdk.RecoverGroupMetadataAsync(
    new ConversationId.Group(groupId), requesterDid,
    lastStateCid, knownMemberDids, knownAdminDids
);
