DChat SDK Identity & Keys

Identity & Keys

DChat identities are self-sovereign a BIP39 seed phrase is the root of trust. No email, no phone, no server-side account creation.

Concepts

Every DChat user has two distinct key layers:

Identity Key (DChatChainKeyPair)
Long-lived Secp256k1 key derived from the seed phrase. Used to sign blockchain transactions (register username) and authenticate to the gRPC server. Never rotated.
Chat Key (ChatKeyPair)
Short-lived Secp256k1 key used exclusively for E2EE message encryption. Rotates automatically based on message count or age for forward secrecy.

DID format

Decentralised Identifiers follow the pattern did:dchat:<username>. A DID links a human-readable username to an on-chain wallet address and a current chat public key.

Seed Phrase Operations

Task<string> GenerateSeedPhraseAsync(int wordCount = 24)

Generates a cryptographically secure BIP39 mnemonic. The phrase is the master secret from which all keys are derived store it offline, never in plaintext on the device.

ParameterTypeDescription
wordCountint12 or 24 words. 24 words = 256 bits of entropy (recommended).
string mnemonic = await sdk.GenerateSeedPhraseAsync(24);
// "abandon ability able about above absent absorb ..."
Task<DChatChainKeyPair> RestoreFromSeedPhraseAsync(string seedPhrase, string derivationPath = "m/44'/118'/0'/0/0")

Deterministically derives a DChatChainKeyPair from a BIP39 mnemonic using the BIP44 derivation path. Produces the same key pair on any device given the same inputs.

ParameterTypeDescription
seedPhrasestringSpace-separated BIP39 mnemonic (12 or 24 words).
derivationPathstringBIP44 path. Default uses Cosmos coin type 118.
DChatChainKeyPair kp = await sdk.RestoreFromSeedPhraseAsync(
    "word1 word2 ... word24");

Key Storage

Key pairs are stored in the platform's secure storage (Android Keystore / iOS Secure Enclave on device). An optional password adds an extra AES-256 encryption layer over the stored bytes.

Task StoreKeyPairAsync(DChatChainKeyPair keyPair, string? password = null)

Persists the identity key pair in secure storage. Pass a password to require it on every retrieval (useful for PIN-protected accounts).

Task<DChatChainKeyPair?> GetKeyPairAsync(string walletAddress, string? password = null)

Retrieves a stored key pair. Returns null if not found. Throws if the password is wrong.

Task<bool> HasKeyPairAsync(string walletAddress)

Checks whether a key pair exists for the wallet address without decrypting it. Useful for onboarding flow gating.

Task DeleteKeyPairAsync(string walletAddress)

Permanently removes the key pair from secure storage. This is irreversible the identity can only be restored from the seed phrase.

Irreversible. Ensure the user has backed up their seed phrase before calling this.

Blockchain Identity

Task<BlockchainIdentityResult> CreateIdentityAsync(string username, string? password = null)

Generates a key pair, stores it securely, requests faucet funding (testnet), and registers the username on-chain in a single call. Returns the identity and transaction hash.

ParameterTypeDescription
usernamestring330 character alphanumeric username. Must be unique on-chain.
passwordstring?Optional password to protect the stored key pair.
Task<bool> CheckUsernameAvailabilityAsync(string username)

Queries the blockchain to check if the username is free. Call before CreateIdentityAsync to give fast user feedback.

Task<BlockchainIdentity?> GetIdentityAsync(string identifier)

Fetches an identity by wallet address or username. Returns null if no matching identity is found on-chain.

Task<string?> GetUserProfileUsernameAsync(string walletAddress)

Resolves a wallet address to its registered username. Returns an empty string (not null) when no profile is found.

Signing & Verification

Task<string> SignMessageAsync(string message, string walletAddress, string? password = null)

Signs an arbitrary message with the identity private key using Secp256k1. Returns a base64-encoded DER signature. Used for gRPC authentication handshakes and ADR-036 off-chain signing.

Task<bool> VerifySignatureAsync(string message, string signature, string publicKey)

Verifies a Secp256k1 signature against a message and public key. Use this to validate contact-request authenticity or any signed protocol message.
