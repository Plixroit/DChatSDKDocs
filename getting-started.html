DChat SDK Getting Started

Getting Started

From zero to sending your first encrypted message in under 5 minutes.

Step 1 Add the SDK

Add a project reference to DChatSDK from your solution:

<!-- .csproj -->
<ProjectReference Include="../DChatSDK/DChatSDK/DChatSDK.csproj" />

For MAUI apps, use the DChatSDK-MAUI wrapper project which registers platform-specific services automatically.

Step 2 Build the client

Use DChatSdkOptions.CreateBuilder() to configure and instantiate the client.

using DChatSDK;
using DChatSDK.Configuration;

// 1. Configure the SDK
DChatSdkOptions options = DChatSdkOptions.CreateBuilder()
    .AddNode(new NodeConfiguration(
        blockchainRpcUrl:  "https://rpc.dchat.network",
        grpcServerUrl:     "https://grpc.dchat.network",
        middlewareApiUrl:  "https://api.dchat.network",
        name:              "mainnet"))
    .SetGroupMemberProvider(myGroupMemberProvider)
    .Build();

// 2. Instantiate the client
IDChatSDKClient sdk = new DChatSDKClient(options);

Step 3 Create or restore an identity

// Generate a fresh seed phrase (store this securely  it's the master secret)
string seed = await sdk.GenerateSeedPhraseAsync(wordCount: 24);

// Derive the key pair from the seed phrase
DChatChainKeyPair keyPair = await sdk.RestoreFromSeedPhraseAsync(seed);

// Store it securely on-device
await sdk.StoreKeyPairAsync(keyPair);

// Register username on the blockchain
BlockchainIdentityResult result = await sdk.CreateIdentityAsync("alice");
Console.WriteLine(result.Identity.WalletAddress); // dc1qxyz...
Never log or transmit seed phrases. Treat them like a private key anyone with the phrase controls the identity.

To restore an existing identity on a new device:

DChatChainKeyPair keyPair = await sdk.RestoreFromSeedPhraseAsync(existingSeedPhrase);
await sdk.StoreKeyPairAsync(keyPair);

Step 4 Connect and stream

// Activate the identity and open the gRPC stream
bool ok = await sdk.ConnectAsync();

// Subscribe to incoming messages
sdk.MessageReceived += (_, msg) =>
{
    Console.WriteLine($"New message from {msg.SenderId}: {msg.Content}");
};

Step 5 Send a message

// Look up the recipient's public key first (fetched during contact exchange)
string recipientPublicKey = "03a1b2...";  // hex-encoded Secp256k1

await sdk.SendTextMessageAsync(
    recipientWalletAddress: "dc1qbob...",
    content:                "Hello, world!",
    senderDid:              "did:dchat:alice",
    recipientDid:           "did:dchat:bob",
    chatId:                 myChatId,
    recipientPublicKey:     recipientPublicKey);
Public key exchange happens automatically during the contact request handshake via ContactRequestMessageData and ContactResponseMessageData. You don't need to implement this yourself the SDK handles it.

Step 6 Disconnect cleanly

await sdk.DisconnectAsync();
sdk.Dispose();

await using IDChatSDKClient sdk = new DChatSDKClient(

Using Dependency Injection

For production apps, register the SDK client in your DI container. This enables better testability, lifetime management, and decoupling from concrete implementations.

// Program.cs or MauiProgram.cs
using Microsoft.Extensions.DependencyInjection;

IServiceCollection services = new ServiceCollection();

// Register SDK options as singleton
services.AddSingleton(sp => DChatSdkOptions.CreateBuilder()
    .AddNode(new NodeConfiguration(
        "https://rpc.dchat.network",
        "https://grpc.dchat.network",
        "https://api.dchat.network"))
    .Build());

// Register SDK client as singleton (recommended)
services.AddSingleton<IDChatSDKClient, DChatSDKClient>();

// For MAUI apps, use the factory:
// services.AddSingleton<IDChatSDKClient>(sp => 
//     DChatMauiClientFactory.Create(sp.GetRequiredService<DChatSdkOptions>()));

Then inject the client in your services, ViewModels, or pages:

public class ChatViewModel
{
    private readonly IDChatSDKClient _sdk;
    
    public ChatViewModel(IDChatSDKClient sdk)
    {
        _sdk = sdk;
        _sdk.MessageReceived += OnMessageReceived;
    }
    
    public async Task SendMessageAsync(string to, string content)
    {
        await _sdk.SendTextMessageAsync(to, content, ...);
    }
}
Why DI? Injecting IDChatSDKClient makes unit testing trivial just swap the real client for a mock implementation. It also lets the container manage the singleton lifetime, ensuring one shared instance across your app.

Handling Incoming Messages

Messages arrive decrypted and ready to use. Subscribe to MessageReceived and route by Payload.Type:

// Subscribe once during initialization
sdk.MessageReceived += OnMessageReceived;

// Handler receives fully decrypted messages
void OnMessageReceived(object? sender, DecryptedMessage msg)
{
    // All messages have these core properties:
    // - msg.SenderId      (wallet address of sender)
    // - msg.ChatId        (conversation identifier)
    // - msg.Timestamp     (when sent)
    // - msg.Payload.Type  (MessageContentType enum)
    // - msg.Payload.Data  (typed payload object)

    switch (msg.Payload.Type)
    {
        case MessageContentType.Text:
        {
            var text = (TextMessageData)msg.Payload.Data;
            ShowTextMessage(msg.SenderId, text.Text);
            break;
        }

        case MessageContentType.Image:
        {
            var image = (ImageMessageData)msg.Payload.Data;
            ShowImage(msg.SenderId, image.Url, image.Caption);
            break;
        }

        case MessageContentType.Video:
        {
            var video = (VideoMessageData)msg.Payload.Data;
            ShowVideo(msg.SenderId, video.Url, video.ThumbnailUrl);
            break;
        }

        case MessageContentType.File:
        {
            var file = (FileMessageData)msg.Payload.Data;
            ShowFileAttachment(msg.SenderId, file.FileName, file.Size, file.Url);
            break;
        }

        case MessageContentType.Location:
        {
            var loc = (LocationMessageData)msg.Payload.Data;
            ShowLocation(msg.SenderId, loc.Latitude, loc.Longitude, loc.Label);
            break;
        }

        case MessageContentType.CallOffer:
        {
            var offer = (CallOfferMessageData)msg.Payload.Data;
            ShowIncomingCall(msg.SenderId, offer.CallId, offer.IsVideo);
            break;
        }

        case MessageContentType.ContactRequest:
        {
            var req = (ContactRequestMessageData)msg.Payload.Data;
            ShowContactRequest(msg.SenderId, req.DisplayName, req.PublicKey);
            break;
        }

        default:
            // Handle unknown or future message types
            LogUnknownMessage(msg);
            break;
    }
}
Thread safety: Events fire on the thread pool. For UI updates, marshal to your UI thread (e.g., MainThread.BeginInvokeOnMainThread in MAUI, or Dispatcher.Dispatch in WPF).

Other useful events

// Track connection state
sdk.ConnectionChanged += (_, connected) =>
{
    if (connected) HideOfflineBanner();
    else       ShowOfflineBanner();
};

// Handle delivery confirmations
sdk.MessageAcknowledged += (_, msg) =>
{
    MarkMessageDelivered(msg.MessageId);
};

// Retry failed messages
sdk.MessageFailed += (_, msg) =>
{
    ScheduleRetry(msg, delay: TimeSpan.FromSeconds(30));
};
